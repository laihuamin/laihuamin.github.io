vue-router深入学习

Aug 2, 2017 | Hits

vue-router是项目中用的挺多的，或许已经会用，但是觉得未必了解，所以打算了解一下他的原理

属性

to
1、类型：string | Location
2、表示目标路由的链接。当被点击后，内部会立刻把 to 的值传到 router.push()，所以这个值可以是一个字符串或者是描述目标位置的对象。

replace
1、类型：boolean
2、默认值：false
3、设置 replace 属性的话，当点击时，会调用 router.replace() 而不是 router.push()，于是导航后不会留下 history 记录。

append
1、类型：boolean
2、默认值：false
3、设置 append 属性后，则在当前（相对）路径前添加基路径。例如，我们从 /a 导航到一个相对路径 b，如果没有配置 append，则路径为 /b，如果配了，则为 /a/b

tag
1、类型：string
2、默认值：‘a’
3、将其渲染出什么标签，如：tag=’li’,最后会渲染出li标签

active-class
1、类型：string
2、默认值：’router-link-active’
3、设置 链接激活时使用的 CSS 类名。默认值可以通过路由的构造选项 linkActiveClass 来全局配置。
exact
1、类型：boolean
2、默认值：false
3、”是否激活” 默认类名的依据是 inclusive match （全包含匹配）。 举个例子，如果当前的路径是 /a 开头的，那么< router-link to=”/a”> 也会被设置 CSS 类名。按照这个规则，< router-link to=”/“> 将会点亮各个路由！想要链接使用 “exact 匹配模式”，则使用 exact 属性：

event
1、类型：string | Array< string >
2、默认值：’click’
3、声明可以用来触发导航的事件。可以是一个字符串或是一个包含字符串的数组。

< router-view> 组件是一个 functional 组件，渲染路径匹配到的视图组件。< router-view> 渲染的组件还可以内嵌自己的 < router-view>，根据嵌套路径，渲染嵌套组件。
name
1、类型：string
2、默认值：’default’
3、如果 < router-view>设置了名称，则会渲染对应的路由配置中components下的相应组件。

其他属性（非 router-view 使用的属性）都直接传给渲染的组件， 很多时候，每个路由的数据都是包含在路由参数中。

Router构造配置

routes
1、类型: Array

mode
1、类型: string
2、默认值: “hash” (浏览器环境) | “abstract” (Node.js 环境)
3、可选值: “hash” | “history” | “abstract”
4、配置路由模式:
⑴hash: 使用 URL hash 值来作路由。支持所有浏览器，包括不支持 HTML5 History Api 的浏览器。
⑵history: 依赖 HTML5 History API 和服务器配置。
⑶abstract: 支持所有 JavaScript 运行环境，如 Node.js 服务器端。如果发现没有浏览器的 API，路由会自动强制进入这个模式。
base
1、类型：string
2、默认值：’/‘
3、应用的基路径。例如，如果整个单页应用服务在 /app/ 下，然后 base 就应该设为 “/app/“。
linkActiveClass
1、类型: string
2、默认值：< router-link-active>
3、全局配置 < router-link> 的默认『激活 class 类名』。
scrollBehavior
1、类型: Function
