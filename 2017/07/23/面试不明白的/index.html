跨域问题

先来说说这个问题为什么会存在？
来源是因为JS中引入了同源策略，就是只能从一个站点获取资源，而不是那些来自其它站点可能怀有恶意的资源，也是为了安全起见。
举个栗子：
一个黑客程序员，利用IFrame标签把真正的银行登陆界面嵌入到自己的网页中，用户利用这个登陆时，就会把input表单中的数据提交到黑客程序员手里，那么后果不堪设想。如何解决跨域问题，笔者这边介绍两种方法
第一种：JSONP
因为做项目的时候jsonp有用过，所以对他的原理还是比较熟的，我就粗略带一下：
jsonp(json with padding)十分官方的解释就是：它是一个非官方的协议，它允许在服务器端集成Script tags返回至客户端，通过javascript callback的形式实现跨域访问。
我自己的理解就是：动态插入< script>标签，然后通过script标签引入js文件，当js加载成功后会执行url指定的函数，获取所需数据。
优点：兼容性好，简单易用，支持浏览器和服务器双向通信。
缺点：只支持GET方式

第二种：CORS
当要请求不再同一站的资源时，然后发起一个跨域的HTTP请求,在同源策略作用下，出于安全考虑，浏览器会限制这个http请求。
CORS(Cross-Origin Resource Sharing)（跨域资源共享）机制允许web应用服务器对跨域访问进行控制，从而保证跨域数据安全传输。浏览器支持在 API 容器中（例如 XMLHttpRequest 或 Fetch ）使用 CORS，以降低跨域 HTTP 请求所带来的风险。
CORS的步骤：
1、在服务端中，在http的head中添加Access-Control-Allow-Origin：*（这里的*代表任意）或Access-Control-Allow-Origin: http: //example.com (具体网址)
2、在客户端中，给一个支持回退功能的XMLHttpRequst的请求
