JS高程（学习笔记）

Jul 23, 2017 | Hits

红皮书白读不厌，百读不精，分享一点学习笔记，持续更新

1、任何涉及NaN的操作都会返回NaN。
2、isNaN函数，参数可以为任意类型，当接收到参数不为数值时，会将其转换为数值，不能转换的会返回true

3、任何String都是可以用length输出长度，但是会带有结束符$
4、Object的每一个实例都具有的方法
⑴Constructor：保存创建该对象的函数。
⑵hasOwnProperty：用于检查给定的属性，当前实例是否存在。
⑶isPrototypeOf：用于检查传入的对象是否是另一个对象的原型。
⑷toString():返回对象的字符串表示。
5、对于非操作符——！

6、关于大于小于操作符—— <、>

7、关于相等操作符—— === == (涉及到 == 的隐式转换)

8、几种特殊情况及结果

9、arguments对象只是类数组（并不是Array的实例），可以用arguments[0]来访问里面的参数。

10、ES中所有参数都是按值传递的，并不是按引用类型传递。

11、
⑴arguments的长度是由传递值的参数决定，而不是由定义决定。
⑵没有传递值的命名参数会自动设置为undefined。
⑶argument的值永远和对应的参数名相对应。

12、ES中不存在函数签名，因此其参数是由包含零或多个值的数组来表示。没有函数签名，函数重载是做不到的。

13、按值访问的：undefined、null、boolean、number、string（基本数据类型）
按引用访问：保存在内存中的对象（引用的类型的值）

14、不能给基本类型的值添加属性(试了一下null也是一样的结果)

15、按值传递的两个值是完全独立的，而按引用传递两个指针指向同一个对象

16、所有函数的参数都是按值传递的这点我们在第10条已经看过了

17、typeOf 和 instanceOf

18、函数的参数可以当作变量来对待。访问规则与执行环境中的其他变量相同。

19、ES5中没有块级作用域（for循环时，i会流出），在ES6中已经添加了

20、不声明直接初始化变量时常见的错误。未用var声明，不但不会报错，而且当执行完add之后，sum会被添加到全局之中

21、查询标识符，是从作用域前端开始，逐级向上查询。其实下面这个例子中，函数并没有color这个变量，这个变量是全局的，所以当自己没有这个变量时他会向上查询，再看下面第二个例子当给函数内部设置一个同名变量，结果就完全不一样了。足可说明先前观点

22、常用的垃圾回收方式：标记清理——会把一个变量进行标记，如进入环境，出环境等，会周期性的清理没用的变量。在全局环境中，没用的变量可以置为null,那样他就会跳出执行环境，从而被清理
不常用的垃圾回收方式：引用计数——他会造成循环计数，从而大量使用内存导致无法回收。（循环引用本身没什么问题，但是序列化的时候就会产生问题）

23、在22点中提到过，解除引用的方式，将变量设置为null,（这种方式适用于全局变量和全局对象属性）。变量会脱离执行环境，被回收。

24、对象字面量是传递大量信息的首选方式，举个例子

25、一些概念：
⑴引用类型：一种数据结构，用于将数据和功能模块绑定到一起
⑵引用类型和类看起来相似，但是不具备传统oop所支持的类和接口
⑶对象是某个特定引用类型的实例

26、访问对象属性时，通常用点表示法，当属性是一个变量时可以用方括号表示法
