web存储

Jul 27, 2017 | Hits

这篇文章主要总结一下cookie、localstorage、sessionstorage这三种常用的存储技术之间的区别

cookie

cookie是最早的一种运用在客户端的数据存储技术，与之相对应的是服务端的session。

cookie存储时有几个限制：

  • cookie存储时，会将数据存储在浏览器的不同域中，而不同的浏览器的存储域几乎是有限的。
  • 每条cookie大致可以存储4kb以下的数据
  1. 设置cookie
  1. 获取cookie中的值
  1. 删除cookie(将有效时间设置为之前的时间)

localstorage(本地存储)

localstorage是html5之后新添加的存储形式，这种存储是永久存储，除非手动删除。localstorage的存储是以（key-value）的形式。主要是为了解决cookie的一些缺点。

cookie缺点：

  • cookie存储数据时，会往服务器发送数据，可是往往有些数据是不需要发送的。
  • cookie有存储大小的限制，在4kb以下cookie是可以存储的，但一旦大于4kb时，cookie会被浏览器自动注销。

localstorage API：

  1. localstorage.setItem(key, value); => 设置key值的value值,存入数据
  2. localstorage.getItem(key) => 得到localstorage中的key值
  3. localstorage.removeItem(key) => 删除localstorage中的key-value
  4. localstorage.removeAll() => 删除localstorage中的所有数据
  5. localstorage.key(index) => 在localstorage中key值的下标

sessionStorage(会话存储)

由于localstorage的永久存储属性，可能并不能满足一些数据的实时存储，所有在html5中还增加了sessionStorage。它存储的数据会在浏览器关闭之后，被全部清除。
