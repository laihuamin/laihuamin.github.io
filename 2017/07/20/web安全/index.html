常见的web安全问题

Jul 20, 2017 | Hits

web安全是很多人企业会关心的话题，也是我们学习时需要知道的，接下来，我将整理一些我所知道的知识，仅供参考。

sql注入原理

就是通过sql指令，将其插入到web表单提交、输入域名、页面输入中，从而欺骗服务器进行恶意的sql指令。
如何防范，看一下几点：
1、永远不要信任用户输入，要对用户输入的进行校验，可以通过正则、限制长度、对单引号和双”-“进行转换。
2、永远不要动态拼装sql，可以使用参数化的SQL或者直接使用储存过程中进行数据库查询（sql拼接除了不安全，还有就是存储效率低）
3、永远不要用root权限登陆，创建一个权力有限的用户。（表示一直没做到过0.0，反正是私服，无所谓了）
4、机密信息永远不要明文存放，加密或hash掉密码和敏感信息。

XSS原理及防范

Xss攻击就是攻击者网web插入html标签或javaScript代码。看似是个安全链接，点进去之后获取用户的cookie信息，当表单提交时，把数据传到攻击者服务器上，而不是原站点服务器。
Xss攻击分为两种，一种是DOM Based XSS漏洞，还有一种是Stored XSS漏洞。理论上，只要对所有可输入地方没有做校验的都会存在漏洞。
