承接上一篇学习笔记

1、字面量是创建对象的首选方式

2、ES中对象有两种属性：数据属性和访问器属性

3、Object.defineProperty()方法接受三个参数：属性所在对象，属性名和一个描述符对象（数据属性(configurable、enumerable、writable、value)的一个或多个）。如果不设置这四个属性默认都是false

4、访问器属性有getter和setter，也可以通过Object.defineProperty来设置

5、设置多个属性时，可以用Object.defineProperties()方法

6、读取属性特性——Object.getOwnPropertyDescriptor()方法

7、当用一个接口创造多个对象时，可以使用构造函数模式

8、任何函数通过new操作符来调用就是将其看成构造函数，而不用new调用与普通函数没什么两样

9、构造函数缺点
⑴当需要封装的属性值过多时，发现构造函数会很累
⑵看一下这个例子中，每次new一个新的对象，对象都会创造一些新的属性，而到sayName这个函数也随着对象的实例化而实例化，ES中的函数就是一个对象，那么其实sayName的功能是一样的，但是他却被多次创建，虽然不同实例中的sayName函数是不一样的。

10、原型模式——这个模式从某些方面弥补了构造函数的不足
⑴什么是原型对象？
每次创建函数，每个函数都会带有prototype属性，这个属性指向函数的原型对象。
而默认情况下所有原型对象都会获取一个constructor的属性，这个属性含有指向prototype属性的函数的指针。

var person1 = new Person();
console.log(person1.proto); // undefined
console.log(Person.prototype.isPrototypeOf(person1)); //true

//例子1
function Person(){
}
Person.prototype.name = ‘lai’;
Person.prototype.age = 29;
Person.prototype.job = ‘student’;
Person.prototype.sayName = function () {
console.log(this.name);
}
var person1 = new Person();
var person2 = new Person();
person1.name = ‘Greg’;
console.log(person1.name); //Greg——来自实例
console.log(person2.name); //lai——来自原型
console.log(person1.sayName()); // Greg
console.log(person1.sayName() === person2.sayName()); //true
